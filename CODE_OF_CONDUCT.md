# Code of Conduct

## Expected Behavior

- Be direct, respectful, and technical.
- Assume maintainers and contributors are trying to improve the project.
- Keep disagreement focused on evidence, tradeoffs, and user impact.
- Share enough context for others to reproduce or evaluate your claim.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Posting secrets, private repository content, or sensitive logs.
- Repeated off-topic comments, spam, or pressure for unpaid support.
- Publicly disclosing security vulnerabilities before maintainers have had a reasonable chance to respond.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; block accounts; or limit participation when behavior harms the project or its contributors.

For security-sensitive issues, use the process in [SECURITY.md](./SECURITY.md). For conduct issues, contact the maintainers through the repository's normal GitHub channels with links and context.
